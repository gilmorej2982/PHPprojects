Forms Mini-Project: Retrieving and using form data (15 pts)

Focus on:

  • Creating a robust, usable form
  • Styling the form using CSS
  • Creating variables from the data sent via POST and using it in scripts
  • New PHP functions: isset(), count(), implode() and the && and ! operators.
  • Using conditional statements and loops.

Description Part I: the form

You have been provided screenshots of a form: one is the unstyled appearance of the form, and the other is what it needs to look like after it is styled.

Create the form as described on miniproject-form.php. Check your unstyled form against the screenshot of the unstyled form, and then begin to style your form. Be sure you have all the correct attributes so that the form VALIDATES. You also need to be sure that you fill in the method and action attributes with the correct information.

You can have any values for the favorite color and TV shows that you want; don't use mine.

Hints

  1. The styled form in our checkboxes exercise has good information about how to mark up the form for styling.
  2. The CSS for that form gives good hints about how to approach styling a form. You can also visit the links on our D2L site.
  3. To validate a form that is .php, you need to EITHER copy and paste the markup into the validator, OR you need to save a copy of the file as a .html file. It won't parse any scripts (so don't use it for your mini-project), but the validator can't check a .php file. You'll need to choose one method to validate your form. (However, you can validate the CSS on a php document).
  4. Use your Forms notes, exercise and powerpoint for hints on writing forms. We've covered it all.

Description Part II: the script

I took a screen shot of what the script processed from MY form. Use it as an example of what you are looking for. Again, you can use whatever colors and TV shows you like as your own checkbox/radio button values. At least 4 each, but you can use more.

The miniproject-script.php has specific instructions about what you should do with your scripts. Follow the instructions. You'll write a series of scripts, but you only need to create variables for the form data once; the variables can be reused in multiple scripts as long as their value isn't re-assigned.

Test your scripts by checking off different numbers of boxes: do it with none checked, with all checked and with some checked. That is what your conditional statement is looking for, and I'll test your script by doing the same thing.

Hints

  1. The Conditional Statement notes, exercise and powerpoint can help you with this.
  2. The Loop notes, exercise and powerpoint can help you with this.
  3. The Checkbox notes, exercise and powerpoint can help you with this.
